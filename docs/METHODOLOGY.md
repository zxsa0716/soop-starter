# 방법론 (Methodology)

본 문서는 숲스타터 시스템의 8단계 분석 모듈에 사용된 방법론을 학술적 형식으로 상세히 기술한다.

## 1. 자연어 결정변수 추출

### 1.1 입력 및 출력

입력은 사용자가 본인의 산촌 진입 의향을 자유 형식으로 기술한 자연어 단문(또는 단락)이다. 출력은 27개 결정 변수로 구성된 JSON 객체이며, 각 변수는 사전 정의된 스키마에 따른다.

### 1.2 추출 방법

거대언어모델(LLM)로 Google Gemini 2.5-flash를 채택한다. 모델 호출 시 시스템 명령어(system_instruction)에 27개 필드의 유형(int, str, list, bool)과 한국어 표기 규칙을 명시하며, `response_mime_type=application/json` 옵션으로 JSON 출력을 강제한다. Few-shot 예시 3개를 시스템 명령어 말미에 포함시켜 일관된 추출 양상을 유도한다.

### 1.3 27개 결정 변수

연령(age), 가구원 수(family_size), 자본금(capital_won), 월 소득(monthly_income_won), 현 직업(current_occupation), 거주 형태(urban_or_rural), 전환 유형(transition_type), 전환 기간(transition_horizon_years), 희망 지역(region_preferences), 관심 임산물(interested_products), 영농 경험(has_farm_experience, farm_experience_years), 주 목표(primary_goal), 부 목표(secondary_goal), 청년 보조 자격(eligible_for_young_subsidy), 40시간 교육 가능성(available_for_40h_training), 이주 준비(ready_to_relocate, ready_within_months), 선호 재배 주기(preferred_product_cycle), 차입 필요(need_loan), 보조금 필요(need_government_subsidy), 위험 성향(risk_tolerance), 탄소상쇄 의향(wants_carbon_offset), 목표 5년 매출(target_5y_revenue_won), 배우자(spouse), 자녀(has_children), 기타 메모(notes).

## 2. 후보 마을 선정

### 2.1 산촌 정의

산림기본법 제3조에 근거한 466개 공식 산촌을 사용한다. 본 연구는 산림청 산촌기초조사 보고서(refs/02) 부록의 산촌 읍면동 목록과 행정안전부 행정경계 SHP의 읍면동 폴리곤을 시도명·읍면동명 기준으로 결합하여 467건의 행정구역 코드(BJCD)를 매핑하였으며, 매핑 불가 5건(주로 도서·산간 특수 행정구역)은 별도 기록하여 추후 보완 대상으로 분류한다.

### 2.2 4축 평가

각 후보 마을에 대해 다음 4축을 0~1 정규화 점수로 산출한다.

- **임업 적합도**: 입지토양도와 임상도에서 추출한 토양형·임상·영급의 가중 평균
- **생활 환경**: 등산로 트레일, 도로중심선, 권역별 POI 밀도의 가우시안 가중 합
- **정책 매칭**: 사용자 프로필과 32개 보조사업 룰의 자격 충족 개수 비율
- **안전성**: 산불 이력(2003~2020) 및 산사태 포인트의 5km 버퍼 내 발생 건수의 역지수

상위 5개 마을을 선정한다.

## 3. 임산물 적합도 예측 (LightGBM)

### 3.1 데이터

국가산림자원조사(NFI) 제7차(2016~2020년) 임분조사표 16,617개 표본점 중 산림 지정(임상코드 1~4) 표본점 12,331개를 학습 모집단으로 사용한다.

### 3.2 입력 변수 (27개)

해발고, 경사, 지형코드, 사면위치, 8방위, 임상코드, 수관밀도, 경급, 영급, 소유, 임종, 지종, 갱신형태, 토양형, 토성 A·B, 암석노출도, 침식상태, 도로 거리, 수관밀도 평균, 그리고 임목조사표 697,451개 수목 레코드를 표본점별로 집계한 우점수종·평균흉고직경·평균수고·최대흉고직경·최대수고·본수.

### 3.3 목표 변수 합성

산림 소득 가이드(refs/04)의 정성적 적합도 룰을 다음 가중치로 정규화하여 0~1 점수로 변환한다.

- 우점수종 일치(0.30) + 임상 일치(0.20) + 해발고 적정성 가우시안(0.20) + 경사 감점(0.15) + 영급 가중(0.15)
- 가우시안 N(0, 0.03²) 노이즈를 추가하여 과적합을 방지

### 3.4 모형

LightGBM 회귀 모형(n_estimators=200, learning_rate=0.05, max_depth=6, num_leaves=31, min_child_samples=20). 5-fold 층화 교차검증(난수 시드 42)을 수행한다.

## 4. 유사 산촌 추천 (그래프 임베딩)

### 4.1 그래프 구조

ForKG-Korea v2 그래프는 노드 729개, 관계 993개로 구성된다. 노드 유형은 공간(Location, 589개), 법령(LegalProvision, 68개), 보조사업(Subsidy, 32개), 데이터원(DataSource, 19개), 임산물(ForestProduct, 10개), 기관(Organization, 4개), 절차(Procedure, 4개), 정책(Policy, 3개)이다.

### 4.2 학습 방법

Node2Vec(Grover & Leskovec, 2016) 알고리즘을 채택한다. 무작위 보행(num_walks=10, walk_len=20, p=1.0, q=0.5)을 생성하고, Word2Vec Skip-gram(vector_size=64, window=10, negative=8, epochs=20)으로 임베딩 행렬을 학습한다. 음성 샘플링(negative sampling)은 정보 노이즈 대비 추정(noise-contrastive estimation, NCE)의 일종으로, 대조 학습(contrastive learning) 계열에 속한다(Mikolov 외, 2013).

### 4.3 평가

전체 993개 관계 중 10%(약 99개)를 홀드아웃하고, 나머지로 임베딩을 재학습한 후 코사인 유사도로 정렬하여 정확도(상위 1, 3, 5, 10개 기준)와 평균 역순위(Mean Reciprocal Rank)를 산출한다.

### 4.4 비교 평가

| 방법 | 차원 | 상위 1 정확도 | 상위 5 정확도 | 상위 10 정확도 | 평균 역순위 |
|---|---|---|---|---|---|
| 단순 원-핫 + 차수 | 11 | 0.000 | 0.000 | 0.000 | 0.003 |
| Spectral Laplacian | 32 | 0.032 | 0.175 | 0.714 | 0.161 |
| **Node2Vec Skip-gram** | **64** | **0.317** | **0.857** | **1.000** | **0.523** |

## 5. 5년 누적 소득 시뮬레이션 (Holt + LightGBM 적층 + 부트스트랩 몬테카를로)

### 5.1 데이터

국가산림자원조사 제5차(2006~2010)·제6차(2011~2015)·제7차(2016~2020) 데이터를 시군구명 기준으로 결합한다. 시군구 코드 체계가 차수별로 상이하므로 시군구 명칭(정규화 후)으로 통합한다. 3개 차수를 모두 보유한 178개 시군구를 검증 모집단으로 사용한다.

### 5.2 단계 1: Holt 선형 추세

각 시군구에 대해 NFI 제5차·제6차 데이터로 Holt 선형 추세 모형(statsmodels의 ExponentialSmoothing, trend='add')을 학습한다. Prophet은 데이터 포인트 3개로는 안정적 추정이 불가능하므로 Holt를 채택한다.

### 5.3 단계 2: LightGBM 잔차 보정

Holt 모형의 학습 잔차를 목표 변수로 하고 시군구별 평균 해발고, 산림 면적 비율을 입력 변수로 하는 LightGBM 회귀 모형을 학습한다(n_estimators=30, learning_rate=0.1, max_depth=3).

### 5.4 단계 3: 적층 예측

NFI 제7차 데이터에 대해 Holt 모형의 추정값과 LightGBM 잔차 모형의 추정값을 합산하여 적층 예측치를 산출한다.

### 5.5 단계 4: 부트스트랩 몬테카를로

사용자 프로필별로 다음 분포에서 1,000회 시나리오를 생성한다.

- 연간 수확량 ~ Lognormal(log(1200), 0.32²) × 성장 배율
- 단가 ~ Lognormal(log(12000), 0.25²)
- 초기 자본 ~ Lognormal(log(8,000,000), 0.20²)
- 연간 운영비 ~ Lognormal(log(2,000,000), 0.15²)

각 시점에서 비관(상위 10%), 중위(50%), 낙관(상위 90%) 백분위수를 산출하여 5년 누적 fan chart를 생성한다.

### 5.6 검증 결과

| 방법 | MAPE 중위값 (178 시군구) |
|---|---|
| Holt 단독 | 93.4% |
| Holt + LightGBM 적층 | 72.5% |
| **개선율** | **+22.3%포인트** |

## 6. 정부지원 자격 자동 판별

### 6.1 룰 인코딩

13종 정책 문서(refs/03·04·05·07·08·09)에서 32개 보조사업의 자격 요건을 YAML 형식의 룰로 정밀 인용한다. 각 룰은 `subsidy_id`, `name_ko`, `amount_won`, `budget_2026_eok`, `eligibility_rules`(필드·연산자·값의 튜플), `cite_source`(refs PDF 경로 및 페이지), `cite_law`(법령 조항), `is_2026_new`(신규 여부) 필드를 가진다.

### 6.2 매칭

사용자의 27개 결정 변수 JSON과 32개 룰의 `eligibility_rules`를 평가하여 충족 여부를 산출한다. 충족된 사업은 5년 일정 표(timeline)로 정렬되어 이번 주 최우선 액션 1개를 추출한다.

## 7. 산촌체류형 쉼터 7개 요건 검증

산촌체류형 쉼터(2026년 산림청 신규 시행)는 다음 7개 요건을 모두 충족해야 신청 가능하다.

1. 거점이 산림기본법 §3 기준 466개 공식 산촌 내일 것
2. 임야 0.1ha 이상 소유 또는 장기 임대
3. 임도 접근 가능 (도로로부터 1km 이내)
4. 신청자 만 60세 미만
5. 산림 기본계획 비저촉
6. 보전산지 비저촉
7. 건축 면적 50m² 이하 (체류 목적 한정)

각 요건에 대한 boolean 평가 결과와 위반 요건 목록을 반환한다.

## 8. 정책 RAG 시스템 (M08)

### 8.1 코퍼스 구축

13종 정책 문서를 PyPDF2 + 한국어 OCR로 텍스트 추출한 후, 400자 단위 청크로 분할한다(50자 중첩). 페이지 마커(===== PAGE N =====) 단위로 1차 분할 후 청크별 메타데이터(source_pdf, page_marker)를 부여한다. 결과는 7,992개 청크로 구성된 JSONL 파일과 BGE-m3(Beijing Academy of Artificial Intelligence, 한국어 SOTA) 1024차원 임베딩을 Chroma 영구 저장소에 색인한다.

### 8.2 검색

사용자 질의에 대해 BGE-m3 임베딩 후 Chroma HNSW 색인으로 상위 5개 청크를 검색한다. BM25 결과와 Reciprocal Rank Fusion(RRF)로 결합 후, bge-reranker-v2-base로 재순위화한다. 상위 청크의 source_id가 ForKG-Korea v2의 LegalProvision 노드로 연결되어 있다면, 해당 노드의 1~2-hop 이웃 노드(Subsidy, Policy)를 추가 컨텍스트로 포함시킨다.

### 8.3 답변 생성 및 인용 강제

검색된 청크와 그래프 이웃 노드를 시스템 명령어에 포함시키고, Gemini 2.5-flash에 다음 가드레일을 적용한다.

- 답변 본문 끝에 `(refs/XX_*.pdf, page N · 법령 조항)` 형식 인용 첨부 강제
- 답변이 인용 형식을 포함하지 않으면 재시도(최대 2회)
- 컨피던스 0.6 미만 시 "해당 정보 부재" 응답

### 8.4 평가

5개 핵심 질의(임업직불금, 산촌 쉼터, 임업후계자, 산림탄소상쇄 KOC, 영양 스마트팜)에 대한 평가에서 출처 매칭 정확도 5/5(100%)를 달성하였다.

## 9. 응답 속도 측정

### 9.1 측정 방법

4개 예시 사용자 × 8개 모듈 × 30회 반복 = 960건의 측정을 수행한다. M01과 M08은 실제 Gemini SDK 호출 시간을 측정하며, M02~M07·M09·M11은 로컬 함수 호출 비용을 측정한다.

| 시나리오 | P50 응답 시간 | 14초 목표 |
|---|---|---|
| 순차 처리 (M01 → 로컬 → M08) | 20.3초 | 초과 |
