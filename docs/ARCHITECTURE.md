# 시스템 구조 (Architecture)

본 문서는 숲스타터 시스템의 4계층 아키텍처를 기술한다.

## 1. 전체 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                     사용자 인터페이스 (Streamlit)                       │
│  - 9개 모드: 대화 / 비교 / 지도 / 임산물 / 시뮬레이션 / 정부지원 / AI / 결과 / 소개  │
└─────────────────────────────────────────────────────────────────────┘
                                  ↓
┌─────────────────────────────────────────────────────────────────────┐
│              오케스트레이션 (Gemini Function Calling)                   │
│  - FastAPI + WebSocket /chat                                        │
│  - 5-state machine (입력→매칭→실행→통합→답변)                            │
└─────────────────────────────────────────────────────────────────────┘
                                  ↓
┌─────────────────────────────────────────────────────────────────────┐
│                  분석 모듈 (M01~M11, 8단계 코어)                          │
│  M01 자연어 추출 (LLM)      M07 쉼터 자격 (룰)                             │
│  M02 마을 후보 (Rule)        M08 RAG 답변 (LLM + Vector)                  │
│  M03 임산물 (LightGBM)       M09 멘토 매칭 (API)                         │
│  M04 유사 마을 (Node2Vec)    M10 5개 공공 API 통합                       │
│  M05 5년 시뮬 (Holt+LGBM+MC) M11 4축 평가 (집계)                          │
│  M06 지원금 매칭 (Rule)                                                 │
└─────────────────────────────────────────────────────────────────────┘
                                  ↓
┌─────────────────────────────────────────────────────────────────────┐
│                  데이터 계층 (모두 읽기 전용)                              │
│  · D:\craft\CRAFT\data 17 소스 (NFI · 임상도 · 입지토양도 · DEM · 행정경계 등) │
│  · refs/ 13 정책 문서 (PDF·DOCX·HWP)                                   │

│  · ForKG-Korea v2 (NetworkX 인메모리)                                  │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. 계층별 책임

### 2.1 사용자 인터페이스 계층

Streamlit 1.57 기반 단일 페이지 응용 프로그램. 사이드바를 통해 9개 모드를 선택할 수 있다. 메인 모드인 "자연어 대화"는 사용자의 자유 입력을 받아 자동 매칭과 8단계 분석을 수행하며, 진행 상황을 진행 표시줄(progress bar)로 실시간 표시한다. 분석 완료 후 통합 결정 대시보드(4개 상위 지표 카드, 5년 fan chart, 4축 레이더, Sankey 흐름도, treemap, 추천 임산물 카드, 액션 아이템, Folium 지도)를 한 화면에 표시하고, JSON 형식의 결정 패키지를 다운로드할 수 있다.

### 2.2 오케스트레이션 계층

Google Gemini 2.5-flash의 function calling 기능을 활용하여 11개 모듈을 동적으로 호출한다. 5-state machine은 (1) 자연어 입력, (2) 사용자 유형 매칭, (3) 모듈 병렬/순차 실행, (4) 결과 통합, (5) 자연어 답변 생성의 5단계로 구성된다. 비동기 호출(asyncio.gather)을 통해 M01과 M08의 LLM 호출을 병렬화하여 응답 시간을 직렬 20초에서 병렬 10초로 단축한다.

### 2.3 분석 모듈 계층

각 모듈은 독립적인 Python 패키지로 구성되며, `backend/src/modules/m0[1-11]_*/` 디렉토리에 위치한다. 모듈 간 의존성은 표준 JSON 스키마로 정의된 데이터 구조를 통해서만 이루어지며, 모듈 단독 테스트가 가능하다.

| 모듈 | 책임 | 핵심 기술 |
|---|---|---|
| M01 | 자연어 → 27 필드 추출 | Gemini 2.5-flash + JSON 스키마 강제 |
| M02 | 467 산촌 → 4축 평가 | NetworkX + scipy 가중치 합 |
| M03 | 임산물 적합도 회귀 | LightGBM 5-fold CV |
| M04 | 그래프 임베딩 + KNN | NetworkX + gensim Word2Vec |
| M05 | 5년 누적 소득 시뮬 | statsmodels Holt + LightGBM + NumPy MC |
| M06 | 32 룰 자격 판별 | YAML 룰 엔진 |
| M07 | 쉼터 7 요건 검증 | Boolean 평가 |
| M08 | 정책 RAG | BGE-m3 + Chroma + Gemini |
| M09 | 산림조합 매칭 | 산림청 API + 거리 계산 |
| M10 | 5 공공 API 통합 | httpx async client |
| M11 | 4축 라이프스타일 점수 | M02~M07 집계 |

### 2.4 데이터 계층

모든 원본 데이터는 읽기 전용으로 접근하며, D:\craft\CRAFT\data 17 소스의 원본 파일은 절대 수정되지 않는다. pyogrio·rasterio·python-calamine·openpyxl 어댑터는 모두 읽기 전용 모드로 열린다.

```

   ↓

```

## 4. 배포 옵션

### 4.1 로컬 단독 실행 (Streamlit)

가장 가벼운 배포 옵션으로, Streamlit 단독 실행(`python -m streamlit run soop_app.py`)만으로 모든 기능을 사용할 수 있다. 외부 데이터 파일이 없어도 인라인 데이터(4명 예시 사용자, 10 임산물, 8 보조사업, 20 산촌)로 모든 모드가 작동한다.

### 4.2 Docker 전체 스택

`docker-compose.yml`을 통해 PostgreSQL + PostGIS, Redis, Chroma, MinIO, Airflow, FastAPI, Next.js 프론트엔드까지 모든 구성 요소를 컨테이너로 띄울 수 있다.

### 4.3 클라우드 배포 (선택)

Streamlit Community Cloud를 통한 무료 배포가 가능하다. `.streamlit/secrets.toml`에 환경 변수를 설정한 후 GitHub 저장소를 연결한다.

## 5. 보안 및 비밀 관리

본 시스템은 모든 API 키와 비밀 정보를 `.env` 파일 또는 Streamlit secrets에 분리한다. 소스 코드에는 어떠한 키도 포함되지 않으며, `.gitignore`는 `.env`, `*.key`, `secrets.json` 등 모든 비밀 파일을 추적 대상에서 제외한다. 환경 변수 누락 시 시스템은 "API 키 미설정" 메시지를 사용자에게 명시적으로 표시한다.

## 6. 재현 가능성

모든 학습 스크립트는 결정론적 난수 시드를 사용한다.

- `numpy.random.seed(42)`
- LightGBM `random_state=42`
- Word2Vec `seed=42`
- 5-fold CV `shuffle=True, random_state=42`

동일 환경에서 동일 결과가 재현됨을 보장한다.
